# PDSA Cycle 2 Audience UI Assessment

## Aim

Improve first-minute comprehension and role-specific usability of the DALY-CARE Atlas for internal DALY-CARE review without changing count logic, source discovery, database querying, payload construction, CSV exports, or privacy safeguards.

## Audience

- PI / senior investigator.
- Data manager / QA.
- Disease researchers for PCD/MM, CLL, lymphoma, and other lymphoproliferative disorders.
- New employees learning safe atlas use.
- Senior clinician / manager reviewers.

## Current Assessment

Cycle 1 already added broad audience routes, trust badges, and MCL/TRIANGLE feasibility caveats. Cycle 2 hardens that layer by adding role-specific routes, a first-fold briefing, a shared trust/status legend, disease entry paths, a data-manager command center, onboarding safe tasks, and a print-friendly clinician/manager briefing.

## Non-Negotiables

- No actual aggregate data are removed.
- No payload arrays, CSV-backed rows, source cards, panels, tabs, audit rows, fallback/reference values, or exports are reduced.
- No patient-level rows, CPR values, identifiers, raw dates, raw free text, or unsuppressed small cells are emitted.
- Feasibility remains descriptive only and must not imply treatment effects, treatment recommendations, safe ASCT/HDT omission, transplant eligibility, validated risk classification, or validated text-derived Ki-67 extraction.
