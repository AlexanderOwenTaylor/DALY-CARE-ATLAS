# PDSA Cycle 2 Plan

## Summary

PDSA Cycle 2 is an additive audience, trust, and presentation pass on top of the existing semantic QA state. It improves internal review navigation while preserving every aggregate data surface.

## Key Changes

- Add a first-fold "Start here — DALY-CARE internal briefing".
- Replace broad route cards with eight role-specific routes.
- Add a global trust/status legend reused in Overview, Clinical Feasibility, Data Dictionary, Source Catalogue, and QA.
- Add disease-specific entry paths for PCD/MM, CLL, lymphoma, and other lymphoproliferative disorders.
- Add Data Manager command center, first 3 safe onboarding tasks, glossary, and clinician/manager print briefing.
- Add preservation snapshots and tests to prevent accidental loss of tabs, panels, rows, output files, export hooks, or payload surfaces.

## Measures

- Preservation snapshot after count is greater than or equal to before count for all tracked surfaces.
- Role-route buttons target valid tabs/subtabs.
- Trust legend labels are visible as text, not color only.
- MCL/TRIANGLE-lite remains descriptive feasibility only.
- No prohibited causal, treatment-recommendation, or patient-level privacy language appears.
