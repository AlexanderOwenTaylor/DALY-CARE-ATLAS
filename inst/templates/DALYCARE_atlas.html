One-page internal briefing

What this atlas is for

A concise interpretation layer for a guided internal review. The full atlas remains available in the detailed tabs, tables, source rows, and exports.

Atlas purpose

DALY-CARE brings aggregate source readiness, semantic lineage, and clinical feasibility signals into one privacy-preserving static review artifact.

What can be used now

  • Source and table discovery.
  • Clinical concept and raw-column signposting.
  • Aggregate feasibility review for study planning.

What needs validation

  • Study-specific definitions, timing windows, and value logic.
  • Candidate evidence such as free-text recovery routes.
  • Disease-specific endpoint and exposure definitions.

What is blocked

  • Causal treatment-effect claims.
  • Clinical treatment recommendations.
  • Unsuppressed patient-level review in the static atlas.
Privacy boundary: the static atlas is aggregate-only. It should not emit patient identifiers, CPR values, raw dates, raw free text, or unsafe row previews.

One click

RStudio Console Run

Open the repository directory in RStudio and source the one-click runner.

source("RUN_DALYCARE_ATLAS.R")

The runner catches errors, stores diagnostics in the session, writes to atlas_runs/, and never calls quit().
