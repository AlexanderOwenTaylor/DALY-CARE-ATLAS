Scope & definitional notes

Compact framing notes for interpreting the atlas counts, scope, and profiling coverage.

Published cohort vs patient table. DALY-CARE covers 65,774 unique lymphoid-lineage cancer patients in the published 2025 paper, while the patient table contains 75,572 individuals because precursor states (such as MBL and MGUS) and broader administrative registrations are also present.
Coverage. 64 datasets catalogued from the DALY-CARE publication. 63 have been profiled with row counts, column inventories, and value distributions. The cartography audit (parts 1–7) resolved 17 initially unloadable datasets via name-variant matching and direct PostgreSQL queries. 1 dataset remains absent: SP_BilleddiagnostikeUndersoegelser_Del2 (imaging free-text descriptions).

Quick start — code examples

Copy-paste recipes from the dalycare R package. These examples show how to load, clean, and analyse DALY-CARE data.

Bootstrap. All sessions begin by sourcing the DALY-CARE package loader. This makes load_dataset(), clean_RKKP_*(), IPI functions, and all helper utilities available.
# Bootstrap the dalycare package
source('/ngc/projects2/dalyca_r/clean_r/load_dalycare_package.R')
help_info()  # Show available functions and examples

RKKP registries

# Load and clean LYFO (lymphoma)
load_dataset('RKKP_LYFO')
LYFO_clean = RKKP_LYFO %>% clean_RKKP_LYFO()

# Kaplan-Meier: DLBCL OS by sex
KM_plot(survfit(
  Surv(time_to_death/365, dead) ~ sex,
  LYFO_clean %>% filter(subtype == 'DLBCL')
))

# Calculate MIPI risk for MCL
MCL = RKKP_LYFO %>%
  clean_RKKP_LYFO() %>% MIPI()
KM_plot(survfit(
  Surv(time_to_death/365, dead) ~ MIPI, MCL
))

Diagnosis cohorts

# Load base tables + ICD10 codes
load_dataset(c('patient', 't_dalycare_diagnoses'))
load_dalycare_icd10()

# CLL cohort from any source
CLL = t_dalycare_diagnoses %>%
  filter_first_diagnosis(ICD10.CLL)

# FL cohort: biopsy-verified only
FL = t_dalycare_diagnoses %>%
  filter(tablename == 't_pato') %>%
  filter_first_diagnosis(ICD10.FL)

KM_plot(survfit(
  Surv(time_dx_death, status) ~ sex, CLL
))

Drug-exposed cohorts

# Young MM patients: full study
MM = t_dalycare_diagnoses %>%
  filter_first_diagnosis(ICD10.MM) %>%
  filter(age_at_diagnosis < 50) %>%
  create_antineoplastic_study()

MM %>% mutate(strata = sex) %>%
  KM_plot_3() # OS + hosp + infection

# CLL venetoclax-exposed cohort
load_ATC_antineoplastics()
CLL = t_dalycare_diagnoses %>%
  filter_first_diagnosis(ICD10.CLL) %>%
  create_antineoplastic_study(ATC.venetoclax)

CLL %>% forest_plot_univariate_os()

Text mining & bias checks

# Pathology text mining
load_dataset('SDS_t_mikro')
SDS_t_mikro %>%
  clean_SDS_t_mikro() %>%
  sample_frac(0.001) %>%
  filter_sentence(text, c('amyloid', 'congo'))

# Evaluate bias
LYFO_clean %>% nrow_npatients()
LYFO_clean$subtype %>% table()
LYFO_clean$time_to_death %>% summary()

  geom_histogram(aes(date_diagnosis))

Medical imaging

Radiology is present in DALY-CARE through three complementary layers: nationwide UX-SKS procedure coding, disease-specific imaging fields in DaMyDa, and SP imaging metadata/report tables in eastern Denmark.

Why this tab matters. The DALY-CARE paper explicitly includes 3.3M radiological examinations and lists medical imaging among the 16 SP EHR modules. This section keeps imaging visible as a first-class atlas layer rather than collapsing it into a placeholder warning.

Why this imaging layer is kept visible

  • Top imaging procedure mix across nationwide sources
  • Visible differentiation between CT, MRI, PET/CT, X-ray, and radiotherapy
  • DaMyDa modality breakdown instead of a generic “imaging exists” note
  • SP imaging coverage (358K examinations across 24,846 patients)

Interpretation

This is not image pixel storage. It is a coverage atlas for imaging events and metadata: radiology procedures in nationwide registers, modality fields in myeloma registry data, and SP-side metadata/report tables that can support later NLP or modality-specific linkage. That distinction is important because the paper also notes that free-text tables, including medical imaging, are handled differently for access and sharing.

SKS procedure codes for treatment identification

To identify treated patients in DALY-CARE, query SDS_t_sksube (40.5M rows) or SDS_procedurer_andre (27.7M rows) using SKS procedure codes. These are the primary nationwide treatment identifiers — they are recorded at the point of care and cover all hospitals.

Database update — April 2026. 20 SDS tables refreshed. 14 new tables added (incl. SDS_doedsaarsag_3, SDS_t_atc, genomics, 5 psychiatry tables). Four renamed: lab_forsker → laboratorieproevesvar, t_tumor → tumor_aarlig, lab_labidcodes → dimlaboratoriekoder, t_vaevsanvend_markoer → vaevregistrering. Core tables (patient, diagnoses_all, adm_medicine) updated; date_death_fu now incorporates LYFO and SDS_doedsaarsag_3. Pending: laboratorymeasurements, t_dalycare_diagnoses, SDS_laegemiddel.
The DALY-CARE server runs on open-source PostgreSQL hosted at the Danish National Genome Center. Data are accessed via R/Python pipelines with indexed queries for fast loading.
